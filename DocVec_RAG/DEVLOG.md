### 关键决策

- 向量数据库基线确定为 Cloudflare Vectorize。
- 原文与中间文本存储确定为 Cloudflare R2 或后端持久层，避免与向量库职责混用。
- 文本处理链路明确为：上传 -> 轻量预处理 -> 两段式分块 -> embedding -> 向量入库 -> 检索生成。
- embedding 约束冻结：索引与查询必须同模型、同维度。
- 模型路由策略冻结：Workers AI 默认，独立模型服务备用。
- 存储策略冻结：R2 主写，失败回退后端持久层，并提供回补能力。

- 完成架构/契约方向收敛，形成阶段计划。
- 对齐前后端职责：前端负责交互和参数，后端负责链路编排。

## 2026-04-19

### 阶段目标

- 完成前端可填 Cloudflare 与生成模型配置入口的方案收口。
- 保证会话级配置安全边界和后端代理边界一致。
- 重写并优化项目说明文档。

### 会话级配置方案（已落地）

- 前端提供高级 API 配置面板，支持填写：
  - Cloudflare Account ID、API Token、Vectorize Index、Namespace、R2 Bucket。
  - Generation Provider、Base URL、API Key、Generation Model ID。
- 配置按 kbName 快照隔离，并存放在 sessionStorage。
- 生效机制为“应用配置”按钮，不在每次业务请求重复提交密钥。
- 业务请求统一携带 X-Client-Session-Id，由后端读取会话上下文。

### 安全边界（已确认）

- 前端不直连第三方模型/Cloudflare API，统一后端代理。
- 密钥仅通过 runtime-config 接口输入。
- 接口读取仅返回脱敏字段，不返回明文密钥。
